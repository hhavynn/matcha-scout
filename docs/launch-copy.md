# Launch Copy

## LinkedIn Post

I built Matcha Scout, a full-stack matcha recommendation app that ranks cafe drinks by your taste preferences.

The fun part is the mix of AI and deterministic scoring: reviews can be parsed into structured taste dimensions, then the backend ranks drinks with transparent match percentages and reasons instead of asking an LLM to make the whole decision.

Tech stack: Next.js, TypeScript, FastAPI, DynamoDB, AWS Lambda/API Gateway, Vercel, Docker Compose, and local Kubernetes with kind.

Live demo: https://matcha-scout.vercel.app

## Casual Reddit / Discord Post

I made a small full-stack app called Matcha Scout. You answer a quick taste quiz, then it ranks fictional matcha cafe drinks by how well they match your preferences.

The backend supports AI-parsed reviews, but the recommendations are deterministic and explainable. I also used it as a DevOps learning project: Vercel frontend, AWS Lambda/API Gateway/DynamoDB backend, Docker Compose locally, and a verified local kind Kubernetes setup.

Demo: https://matcha-scout.vercel.app

## 30-Second Spoken Pitch

Matcha Scout is a product engineering project I built around a simple idea: matcha reviews are subjective, so the app turns taste language into structured data and uses that to recommend drinks. The frontend is Next.js on Vercel, the backend is FastAPI on AWS Lambda behind API Gateway, and DynamoDB stores the cafe, drink, review, and taste-profile data. I kept the recommender explainable with deterministic scoring, added Gemini-compatible parsing for reviews, and validated local development with Docker Compose plus a kind Kubernetes workflow.
