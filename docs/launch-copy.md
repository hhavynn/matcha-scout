# Launch Copy

## LinkedIn Post

I built Matcha Scout, a full-stack matcha recommendation app that ranks cafe drinks by your taste preferences.

The fun part is the mix of AI and deterministic scoring: reviews can be parsed into structured taste dimensions, then the backend ranks drinks with transparent match percentages and reasons instead of asking an LLM to make the whole decision.

Tech stack: Next.js, TypeScript, FastAPI on Vercel Functions, Neon PostgreSQL, Docker Compose, and local Kubernetes with kind.

Live demo: https://matcha-scout.vercel.app

## Casual Reddit / Discord Post

I made a small full-stack app called Matcha Scout. You answer a quick taste quiz, then it ranks verified matcha drinks with real community taste data.

The backend supports AI-parsed reviews, but the recommendations are deterministic and explainable. The frontend and FastAPI backend run on Vercel, Neon PostgreSQL stores the data, and Docker Compose plus kind support local development and demos.

Demo: https://matcha-scout.vercel.app

## 30-Second Spoken Pitch

Matcha Scout is a product engineering project I built around a simple idea: matcha reviews are subjective, so the app turns taste language into structured data and uses that to recommend drinks. The frontend is Next.js on Vercel, the backend is FastAPI on Vercel Functions, and Neon PostgreSQL stores café, drink, review, and taste-profile data. I kept the recommender explainable with deterministic scoring, added Gemini-compatible parsing, and validated local development with Docker Compose plus a kind workflow.
